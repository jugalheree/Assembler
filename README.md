## ğŸ§  Key UX Decisions

### 1. Guided flow instead of direct browsing

To reduce confusion and poor hiring decisions, users are guided to structure their needs before seeing any talent.

### 2. City-first execution context

Hardware execution is physical and coordination-heavy. Anchoring the journey around city selection increases feasibility, accountability, and trust.

### 3. Recommendation before discovery

Presenting a system recommendation (cluster vs expert) gives users a clear starting point and reduces cognitive overload.
